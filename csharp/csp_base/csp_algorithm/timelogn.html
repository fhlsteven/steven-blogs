# 时间复杂度 O(log n) 意味着什么？

  • 原文地址：What does the time complexity O(log n) actually mean?open in new window

事先了解算法的复杂性是一回事，了解其背后的原因是另一回​​事。

无论你是计算机科学专业的毕业生，还是想要有效处理优化问题的人，如果你想利用你的知识解决实际问题，这是你必须理解的东西。

先从简单直观的 O(1)O(1)O(1) 和 O(n)O(n)O(n) 复杂度说起。O(1)O(1)O(1) 表示一次操作即可直接取得目标元素（比如字典或哈希表），O(n)O(n)O(n) 意味着先要检查 nnn 个元素来搜索目标，但是 O(logn)O(logn)O(logn) 是什么意思呢？

你第一次听说 O(logn)O(log n)O(logn) 时间复杂度可能是在学二分搜索算法的时候。二分搜索一定有某种行为使其时间复杂度为 lognlog nlogn。我们来看看是二分搜索是如何实现的。

因为在最好情况下二分搜索的时间复杂度是 O(1)O(1)O(1)，最坏情况（平均情况）下 O(logn)O(logn)O(logn)，我们直接来看最坏情况下的例子。已知有 16 个元素的有序数组。

举个最坏情况的例子，比如我们要找的是数字 13。

每次和中间元素比较都会使搜索范围减半。

所以为了从 16 个元素中找到目标元素，我们需要把数组平均分割 4 次，也就是说，

16×(12)4=116 \times (\frac{1}{2})^4 = 116×(21​)4=1

最终结果

现在来看看「对数」的定义：

为使某数（底数）等于一给定数而必须取的乘幂的幂指数。

也就是说可以写成这种形式
